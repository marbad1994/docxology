# Licensing Notes

## Current state

The repository currently documents behavior and ships an archived X.Org corpus, but it does not yet present a single obvious root license file for the app itself.

## What that means

Before wider public distribution, the application code should have an explicit project license added at the repository root so users and contributors know the terms clearly.

## Archived X.Org content

The archived X.Org documentation keeps its own original notices and historical licensing context. Treat that corpus as separate from the app code when communicating release terms.

## Recommended next step

Add a root `LICENSE` file and then update this page so it names the exact license rather than describing the gap.

## Why this matters

If the project wants outside contributors, downstream users, or packaged releases, unclear licensing creates friction immediately.
