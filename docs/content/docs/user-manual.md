# User Manual

## Open the documentation

Use the documentation overview as your starting point. From there you can open categories, jump into documents, or switch into editing flows.

## Create structure first

A good workflow is:

1. Add a category
2. Give it a title, icon, and short description
3. Add documents inside that category
4. Set document order and decide whether each page appears in the published index

## Write documents

Each document supports a simple Markdown-like authoring flow with:

- headings and subheadings
- lists
- code blocks
- links
- tables

The editor preview is there to help you write. The published view is where the final design should feel more refined.

## Search and navigation

Use global search to jump across the workspace. Inside a document, use the sidebar and the On this page section to move through headings.

## Importing files

Import Markdown, plain text, or HTML-like documents when you want to start from existing material instead of writing from scratch.

## Publishing mindset

When the docs are ready for release, the reader experience should be clean and focused. Authoring controls should stay out of the published presentation.
