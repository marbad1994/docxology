# Structuring Categories and Pages

## Categories should stay broad

Categories are the top-level navigation units in the sidebar and on the overview screen. They should represent broad themes, not tiny fragments.

Good examples:

- Getting Started
- Guides
- Reference
- Project and Community
- Legal and Release

## Pages should stay focused

A page should answer one question well. If a page starts mixing onboarding, contributor policy, release notes, and legal detail, it probably wants to be split.

## Use ordering intentionally

Every category and every document should have an intentional order. That keeps the documentation predictable and avoids needing drag-and-drop for basic organization.

## Keep summaries useful

A document summary should help somebody decide whether to open the page. One short sentence is usually enough.

## Use headings for depth, not for everything

Inside a page, use headings and subheadings to create the in-page tree. Save new pages for real topic boundaries.
