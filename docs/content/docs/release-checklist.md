# Release Checklist

## Documentation checks

- category ordering is intentional
- page ordering is intentional
- summaries are filled in
- hidden pages are hidden on purpose
- links point to the right destinations

## UX checks

- published view does not expose editing controls
- sidebar states are clear
- On this page navigation feels correct
- tables and code blocks still look consistent with the product

## Content checks

- contributor guidance matches current practice
- licensing language matches the repository reality
- archived X.Org links still resolve

## Final reminder

A release build should feel calm and trustworthy. If anything still feels like editor scaffolding, it probably needs another pass before shipping.
