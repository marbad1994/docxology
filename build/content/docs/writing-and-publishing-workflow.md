# Writing and Publishing Workflow

## Draft in categories

Start with category descriptions and page titles before polishing prose. Clear structure makes later writing much easier.

## Write in plain language

This app is for developer docs, so the writing should be direct and practical. Prefer clear instructions over marketing language.

## Preview while you work

Use the preview to catch broken tables, awkward heading depth, and layout issues early.

## Review from the published side too

A page can look fine in the editor preview and still feel weak in the published reading flow. Always read it in the document view before calling it done.

## Release checklist

- category names are stable
- ordering makes sense
- summaries are filled in
- hidden pages are hidden intentionally
- links work
- tables still read well on smaller screens

## Keep release mode clean

The published experience should feel like a finished documentation site, not like an editor with a few buttons hidden.
